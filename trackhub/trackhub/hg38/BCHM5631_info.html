When you are grouping tracks using compositeTrack, multiWig or superTrack settings, adding html pages for each set will be helpful in explaining to users what your grouping means (done by adding a line like html examplePage for the track stanza you wish to annotate).

It is suggested to always provide some form of contact information, if you will have public users.

Read more about the html page option in the Track Database Definition Document by clicking here and the html link. You will see we recommend adding sections titled Description, Display Conventions and Configuration, Methods, Credits, and References.

You can read more about suggested layouts for hubs that are shared on the UCSC public hubs page here.
